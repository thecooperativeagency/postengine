QUICK START
════════════════════════════════════════════════════════════════════════════════

1. Backend Setup (5 min)
   cd backend
   npm install
   cp .env.example .env
   # Edit .env with DATABASE_URL
   npm run db:migrate
   npm run db:seed
   npm run dev

2. Frontend Setup (5 min)
   cd ../frontend
   npm install
   npm run dev

3. Access
   Frontend: http://localhost:3000
   Backend: http://localhost:3001
   API: http://localhost:3001/api/*

TROUBLESHOOTING QUICK REFERENCE
════════════════════════════════════════════════════════════════════════════════

Problem                          Solution
────────────────────────────────────────────────────────────────
Dashboard shows no data          Check: http://localhost:3001/health
Frontend can't reach API         Check: vite.config.js proxy config
No posts showing                 Get X API key, add to .env, restart
WebSocket not connecting         Check backend URL in frontend env
Database errors                  Check DATABASE_URL format, DB running
Alerts not triggering            Check alert rules in alertEngine.js
Slow performance                 Add Redis caching, optimize queries
